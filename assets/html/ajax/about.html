Resumo

Olá, visitante! Seja bem-vindo ao meu pequeno projeto, onde eu dei um pequeno mergulhei no vasto universo do Rick and Morty utilizando tecnologias web. Meu nome é Eduardo, e este projeto representa minha jornada no desenvolvimento web, especialmente focado em aprender jQuery com Ajax, integrado com uma API externa!

Quem sou Eu?

Olá sou um desenvolvedor Full Stack em ascensão, construindo uma base solida em HTML, CSS, JavaScript e outras tecnologias front-end como jQuery, Ajax, React.js também estou mergulhando nas maravilhas do back-end com Node.js, MongoDB, MySQL. E também Estou empenhado em aprimorar minhas habilidades em entender código legado kk. Também sou fascinado pelo mundo Linux e apaixonado pela flexibilidade do Docker.

Por que você construiu isso?

Eu desenvolvi esse projeto no intuito de consolidar todos os conhecimentos que obtive estudando jQuery, o conhecimento só se torna valioso quando é aplicado na prática. A teoria e o entendimento conceitual são fundamentais, mas a verdadeira maestria e compreensão profunda geralmente vêm da experiência prática! 

Quais tecnologias foram utilizadas?

  • jQuery: Facilita a manipulação do DOM e interação com a API de forma eficiente.

  • API do Rick and Morty: Fonte de dados rica em informações sobre a série

  • JAVASCRIPT: Está responsável por tornar a interface do usuário dinâmica, interativa e capaz de se comunicar com uma API externa para obter dados em tempo real. Ele desempenha um papel crucial na experiência do usuário, permitindo uma navegação fluida e a exibição de informações relevantes.

  • HTML: Apenas uma alguns elementos básicos para ser a estrutura base do projeto, pois a maioria dos elementos está sendo criado pelo jQuery!

  • CSS: Básico, pois o intuito do projeto não foi mostrar minhas habilidades em CSS, mas sim em manipulação do jQuery!

Quais foram as maiores dificuldades do projeto?

Os maiores desafios do projeto foram a navegação entre as pages, pois a criação delas em jQuery não é muito intuitiva igual ao react.js, além de são ser aconselhado criar SPA em jQuery, navegação entres os personagens e impedir que os elementos acabem se clonado no dom, além de apenas subsistir os anteriores
